## 購買管理SaaS：全機能要件一覧（MAXレベル・統合ドラフト）

### 0. 基盤（全体共通）

SaaS全体を支える非機能要件および共通機能。

- **0.1 組織・権限管理（マルチテナント対応）**
    
    - 組織・部門・プロジェクト階層管理（ツリー構造）
        
    - ユーザーアカウント管理（招待、凍結、削除）
        
    - ロールベースの権限設定（管理者、購買担当、依頼者、承認者、現場担当など）
        
    - 機能別アクセス制御（「見積作成は可能だが、承認は不可」など）
        
    - データ別アクセス制御（「Aプロジェクトの購買情報のみ閲覧可」など）
        
    - 承認権限・経路設定（金額、品目カテゴリ、部門別など柔軟なルール設定）
        
- **0.2 ユーザー・監査ログ（証跡管理）**
    
    - **証跡の完全保全（Immutable Log）**: 全ての操作（作成、更新、削除、閲覧）履歴を記録
        
    - 承認履歴（誰が、いつ、どの情報に基づき承認したか）の完全追跡
        
    - ログイン履歴、IPアクセス制限
        
    - 監査ログのエクスポート機能
        
- **0.3 通知・リマインドエンジン**
    
    - イベントドリブン通知（タスク発生、差戻し、承認依頼、設変発生、差異検知）
        
    - 期限ベースのリマインド（見積回答期限、納期遅延アラート、承認期限）
        
    - 通知チャネル設定（システム内通知、メール、Slack/Teams連携）
        
    - ユーザー別通知ON/OFF設定
        
- **0.4 マスタ管理**
    
    - **品目マスタ**: 品番、品名、カテゴリ、仕様、図面番号、単位
        
        - **品番バージョン属性管理**: 品番の世代（Rev. A, Rev. B）と変更履歴を管理
            
    - **取引先マスタ（サプライヤ）**: 基本情報、担当者、取引条件、契約情報、評価スコア
        
    - **取引先マスタ（納入先）**: 拠点、倉庫、住所管理（建設業の現場対応）
        
    - **勘定科目・部門マスタ**: 会計連携用のコード管理
        
    - **通貨・税区分マスタ**: 国内・海外取引対応
        
    - 各種マスタのCSVインポート／エクスポート
        
- **0.5 API / 外部連携ハブ**
    
    - **Webhook（イベント通知）**: 設変発生時、承認完了時などに外部へ通知
        
    - **RESTful API（公開）**: 外部システム（ERP, 会計等）からのデータ操作
        
    - **連携アダプタ（標準）**:
        
        - PLM/CAD/PDM連携（図面、BOM、設変情報取得）
            
        - 会計ソフト連携（freee, 弥生, 勘定奉行など）
            
        - ERP連携（生産管理、在庫管理）
            
        - コミュニケーションツール連携（Slack, Teams）
            
- **0.6 ドキュメント管理（証憑ストレージ）**
    
    - 多様なドキュメント格納（PDF, CAD, Office, 画像）
        
    - **図面ハッシュ・バージョン照合**: 同一ファイル名の異なるバージョンを検知
        
    - **差分サムネイル比較**: 新旧図面の変更点を視覚的にプレビュー
        
    - 全証憑（見積書、発注書、請求書、検収書）と案件の紐付け管理
        

### 1. 購買依頼・要件起票

設計・現場からの調達要求を構造化し、購買部門が動ける状態にする。

- **1.1 構造化起票テンプレート**
    
    - 調達目的（試作、量産、保守）、前工程、用途の選択式入力（必須化）
        
    - 品目カテゴリ別カスタムフォーム（例：金型と標準品で入力項目を変更）
        
- **1.2 図面・仕様添付と形式・バージョン確認**
    
    - **PLM連携による最新版図面自動呼出**（品番・図番入力時）
        
    - 図面版数（バージョン）の自動照合・差分プレビュー
        
    - 関連仕様書・管理要求文書の添付
        
- **1.3 数量・納期の確度管理**
    
    - 数量ステータス管理（「見込み」「確定」「変更可能性あり」）
        
    - 希望納期入力（カレンダー）
        
    - **自然言語による納期入力支援**（例：「来週金曜」→日付に自動構造化）
        
- **1.4 起票時点の承認ワークフロー**
    
    - 依頼内容（金額、重要度）に応じた承認ルート自動設定
        
    - Slack/メール連携による簡易承認UI（URLクリックで承認）
        
    - 承認・差戻し（コメント必須化）
        
- **1.5 BOM（部品表）参照・取り込み**
    
    - **PLM/ERPからのBOM連携**（API経由）
        
    - **Excel/CSVのBOMアップロード・取込**
        
    - BOM展開による必要部品の一括依頼起票
        
    - **部品マスタ正規化支援**（例：Excelの「ネジ M3」をマスタの「S-M03」に紐付け支援）
        
- **1.6 設計変更との整合性チェック**
    
    - **BOM変更トリガーでのアラート**: 依頼中・見積中の品目にBOM変更が発生した場合、担当者にアラート
        
    - **自動差分抽出**: 旧依頼内容と新BOMの差分（追加、削除、変更）をハイライト
        

### 2. 見積依頼（RFQ）・引合

購買依頼に基づき、サプライヤへ見積を依頼し、回答を回収する。

- **2.1 依頼内容の整備**
    
    - 購買依頼からRFQへのデータ自動引継ぎ（転記不要）
        
    - 図面・BOM・仕様書の自動添付（1.2のファイル）
        
    - 見積依頼項目テンプレート（納期形式、分納可否、必須回答項目など）
        
- **2.2 RFQテンプレート生成**
    
    - サプライヤマスタと連動し、取引条件・通貨・支払サイトを自動反映
        
    - 案件に応じたテンプレート（標準品用、カスタム品用）選択
        
- **2.3 送付チャネル統合**
    
    - システムからのメール一括送信（サプライヤポータルへのリンク）
        
    - FAX送信API連携（FAXが必要な旧来のサプライヤ対応）
        
    - 送信ログ・ステータス管理（送信済、開封済、回答済）
        
- **2.4 回答期限管理**
    
    - 回答期限の設定と自動リマインド通知（サプライヤ向け）
        
    - **RFQステータスダッシュボード**: 案件別の進捗（未回答、回答済、期限切）を可視化
        
    - フォロー（催促）履歴の記録
        
- **2.5 回答回収（マルチチャネル）**
    
    - **サプライヤポータル（Webフォーム）回答**: 最も望ましい構造化データ回収
        
    - **AI-OCRによる見積書読み取り**: PDFやFAXで届いた見積書をAIがデータ化
        
    - **項目マッピング支援**: AIが読み取った項目（例：「Price」）をシステム項目（「単価」）に紐付け
        
    - **見積比較表の自動生成**: 回収したデータを横並びの比較表に自動整形
        
- **2.6 AIによる推奨サプライヤ提示**
    
    - 品目カテゴリ、過去の実績（納期遵守、品質）、価格傾向をAIが学習
        
    - RFQ作成時に「推奨サプライヤ」をサジェスト
        

### 3. 見積比較・決裁

回収した見積を評価し、発注先を選定する。

- **3.1 多軸比較UI（比較表）**
    
    - 価格（単価、総額、運賃込/別）、納期、支払条件でのソート・比較
        
    - **AIによる条件差異ハイライト**: （例：「A社は分納不可」「B社は保証期間が短い」）
        
    - **過去実績の表示**: 比較表に「過去のトラブル履歴」「納期遵守率」を併記
        
- **3.2 稟議書の自動生成**
    
    - **ワンクリック稟議作成**: 比較表、選定理由、添付図面を自動で集約
        
    - 承認経路の自動設定（金額ルールに基づく）
        
    - 電子印・電子署名連携
        
- **3.3 モバイル承認・ステータス管理**
    
    - 承認者への期限付き通知
        
    - スマホ/タブレットからの簡易承認・差戻し
        
    - 承認プロセス全ステップのログ化
        
- **3.4 選定の透明性確保（説明責任）**
    
    - サプライヤスコアリング（定量評価）の表示
        
    - **選定理由の記録（必須化）**: 「なぜこのサプライヤを選んだか」のログを保存
        
    - （関連機能：15.2 AI交渉コーチング）
        

### 4. 発注書作成・送付

決裁された内容に基づき、法的拘束力のある発注書（PO）を発行する。

- **4.1 発注品・条件の自動反映**
    
    - 決裁された見積データを（転記ミスなく）発注書に自動反映
        
    - 仕様、単価、数量、納期、納入場所、支払条件の確定
        
- **4.2 テンプレート選定・契約条件反映**
    
    - サプライヤ別（または自社標準）の発注書フォーマット自動適用
        
    - **契約DB（SRM）と連携**し、最新の一般取引条件を自動添付
        
- **4.3 マルチチャネル送付とログ管理**
    
    - サプライヤポータルへの発行（推奨）
        
    - メール（PDF添付） / FAX-API送信
        
    - 送信ログ、サプライヤによる「受注確認済」ステータス管理
        
- **4.4 自動発番・履歴・トレーサビリティ**
    
    - 発注番号（PO番号）の自動採番（ルール設定可）
        
    - 発注履歴のバージョン管理（変更・修正履歴の追跡）
        

### 5. 設計変更・再発注対応

発注後に発生する「設計変更（設変）」という最大のリスクイベントに対応する。

- **5.1 設変通知の受領（トリガー）**
    
    - **PLM/CAD/PDMからのWebhook連携**: 設変（出図）をリアルタイムに検知
        
    - 手動による「設変発生」起票（連携未対応の部門向け）
        
- **5.2 影響発注の自動特定**
    
    - **新旧BOM差分比較**: 変更されたBOMと「発注済」「見積中」の案件を自動突合
        
    - **影響部品の自動ハイライト**: 担当者に「この発注に影響あり」と即時アラート
        
- **5.3 サプライヤ再交渉ログ**
    
    - サプライヤへの修正内容通知テンプレート（差分明記）
        
    - 条件変更（価格、納期）の交渉履歴（チャット・コメント）を案件に紐付け
        
- **5.4 修正発注のバージョン管理**
    
    - 既存発注の「キャンセル」または「修正」
        
    - 修正発注書（Rev.2）の発行（変更履歴、差分を明記）
        
    - キャンセル理由の記録
        
- **5.5 特急・キャンセル稟議**
    
    - 設変に伴う特急費、キャンセル料、金型修正費の申請フロー
        
    - **影響金額（損失）の可視化**と専用稟議テンプレート
        

### 6. 納期回答・フォロー

発注したモノがいつ届くかを追跡・管理する。

- **6.1 納期入力ポータル（サプライヤ向け）**
    
    - サプライヤが発注書に対し、納期回答をWebフォームから入力
        
    - （発注書受領確認と納期回答の一元化）
        
    - **納期回答の可視化ボード**: 全発注の納期ステータスを一覧化
        
- **6.2 催促・アラート（遅延防止）**
    
    - 納期未回答のサプライヤへの自動催促（リマインド）
        
    - 回答納期が要求納期より遅れる場合の「遅延アラート」
        
    - **AIによる遅延リスク予測**: サプライヤの過去実績から遅延リスクを警告
        
- **6.3 納期確定の内部配信**
    
    - 確定（または変更）された納期を、関連部門（依頼元、工程管理）へ自動通知
        
    - **工程連携API**: 納期情報を生産管理システム（ERP）へ連携
        
    - 納期変更ログの保存
        

### 7. 納品・受入・検収

現物が届き、発注通りかを確認し、資産として受け入れる。

- **7.1 納品予定共有（ASN）**
    
    - サプライヤがポータルから出荷日・納品予定日・便名を事前入力
        
    - 受入現場の「搬入カレンダー」と連携
        
- **7.2 荷受・検品（現場機能）**
    
    - **モバイルスキャン（スマホ/タブレット）**: 納品書のバーコード/QRをスキャン
        
    - 発注データと現物の「品番」「数量」を即時照合
        
    - 受入実績のリアルタイム同期
        
- **7.3 検収記録と三点照合（1点目：納品）**
    
    - 検品OK（＝検収）のデジタル記録（電子署名・承認）
        
    - 発注データ vs 納品（検収）データの突合
        
    - （関連：8.2 三点照合）
        
- **7.4 納品異常報告（例外処理）**
    
    - 数量不足、破損、仕様不一致の報告フォーム
        
    - **写真添付機能**: 現場がスマホで撮影した不良品画像を添付
        
    - 返品・再手配ワークフローの自動起票
        

### 8. 請求処理・支払（P2P）

サプライヤからの請求書を処理し、支払を完了させる。

- **8.1 請求書受領・データ化**
    
    - サプライヤポータルからの請求書アップロード（推奨）
        
    - **AI-OCRによる請求書読み取り**: 郵送/FAX/メールPDFの請求書を自動データ化
        
    - 受領箱（キュー）での一元管理
        
- **8.2 三点照合と差異ワークフロー**
    
    - **「発注書」 vs 「検収書」 vs 「請求書」の自動突合**（品番、数量、単価、総額）
        
    - **差異自動検知・アラート**: 1円、1個でも異なれば担当者にアラート
        
    - 差異許容ルールの設定（例：±100円までは自動承認）
        
    - 差異発生時の調査・修正ワークフロー（購買、経理、現場への確認依頼）
        
- **8.3 支払予定作成・債務登録・会計連携**
    
    - 照合OK（＝支払承認）となった請求書の支払予定データを自動作成
        
    - **会計ソフト（freee/弥生/奉行/SAP等）への仕訳データ連携**（API/CSV）
        
    - 債務登録（買掛金計上）
        
- **8.4 支払承認・送金・履歴管理**
    
    - 経理部門による支払承認ワークフロー
        
    - FB（ファームバンキング）データ作成、または銀行API連携（送金指示）
        
    - 支払ステータス管理（未払、支払済）と消込
        

### 9. 実績集計・分析（Spend Cube / BI）

蓄積された全データを分析し、戦略的な意思決定（攻めの調達）に活用する。

- **9.1 支出ダッシュボード（Spend Cube）**
    
    - 「いつ、誰が、何を、どこから、いくらで」買ったかを全社横断で可視化
        
    - 品目カテゴリ別、部門別、サプライヤ別、プロジェクト別の支出分析
        
    - **マーベリック・スペンド（ルール外購買）の特定**
        
- **9.2 KPI・パフォーマンス分析**
    
    - **コストKPI**: 単価推移（インフレ/デフレ）、コスト削減額、特急費発生額
        
    - **納期KPI**: 納期遵守率（OTD）、購買リードタイム（依頼～発注）
        
    - **品質KPI**: 受入不良率（PPM）、クレーム件数
        
- **9.3 設変起点のKPI算出**
    
    - 設計変更の発生回数、設変起点の再発注件数、影響金額（損失）の集計
        
    - 設変対応リードタイムの可視化
        
- **9.4 サプライヤ・スコアリング**
    
    - 納期遵守率、品質、コスト、対応（レスポンス）を自動でスコア化
        
    - サプライヤ別評価レポート
        

### 10. サプライヤ管理（SRMライト）

取引先情報を一元管理し、関係性を強化する。

- **10.1 基本情報・契約管理**
    
    - サプライヤ基本情報（住所、連絡先、担当者、銀行口座）
        
    - **契約書データベース**: 契約書PDFの保管、契約期間・更新日のアラート
        
    - 取引条件（支払サイト、通貨）のマスタ管理
        
- **10.2 パフォーマンス記録**
    
    - （9.4 サプライヤ・スコアリングの元データ）
        
    - 納期遅延、品質トラブル、クレームの履歴をサプライヤに紐付けて記録
        
- **10.3 コミュニケーション・交渉履歴**
    
    - **チャット/議事録の一元ログ**: サプライヤとの交渉メモ、問い合わせ履歴を管理
        
- **10.4 リスク・ESG管理**
    
    - 取引停止・再開ステータス管理（警告累積、自動アラート）
        
    - （初期企画書より）**Scope 3（排出量）データ収集**: サプライヤからESG関連データを収集するフォーム・DB
        

### 11. サプライヤ／社外向けポータル

サプライヤがSaaSにアクセスするための専用インターフェース。

- **11.1 見積対応（RFQ）**: （2.5 関連）
    
    - 見積依頼一覧、RFQ詳細確認、Webフォームからの見積回答
        
- **11.2 受注・納期対応（PO）**: （4.3, 6.1 関連）
    
    - 発注書（PO）一覧、受注確認、納期回答入力
        
- **11.3 納品・請求対応**: （7.1, 8.1 関連）
    
    - 納品予定（ASN）入力、請求書アップロード・作成
        
- **11.4 情報管理**: （10.1 関連）
    
    - 自社（サプライヤ）基本情報・口座情報のセルフメンテナンス
        
    - 共有ドキュメント（契約書、図面）の閲覧、Q&A掲示板
        

### 12. 設計連携（E-Chainブリッジ）

「設計」と「購買」の壁を壊す、このSaaSの最重要戦略機能。

- **12.1 PLMからのE-BOM取り込み**: （1.5 関連）
    
    - 設計部品表（E-BOM）の構成、属性、図面情報を同期
        
- **12.2 CAD/PDMの図面受領・バージョン管理**: （0.6, 1.2 関連）
    
    - 出図（リリース）と連動した最新図面の自動取り込み
        
- **12.3 出図予定管理**
    
    - ガントチャートに依存しない「出図予定日」の管理と進捗確認
        
    - 出図遅延と購買手配（LT）の連動アラート
        
- **12.4 設計（図面）変更依頼の起票・伝達**
    
    - 購買・現場・サプライヤから設計への「変更依頼（DR）」を起票する機能
        
- **12.5 S-BOM（サービスBOM）連携**
    
    - 保守・メンテナンス用部品表（S-BOM）と連携し、保守トレーサビリティを強化
        
- **12.6 出図・設変と手配の「つながり」可視化**
    
    - （5.2, 9.3 関連）
        
    - どの設変が、どの発注に、どう影響したかを追跡・可視化するダッシュボード
        

### 13. 間接材／役務（MRO）対応

直接材（BOM部品）以外の、あらゆる調達に対応する。

- **13.1 申請型購買ワークフロー**
    
    - （1. 購買依頼 とは別の）間接材・役務用の申請フォーム
        
    - 部門起票 → 上長承認 → 購買レビュー → 発注 の専用フロー
        
- **13.2 フリー入力型の品目支援**
    
    - 品番がない（フリー入力）品目のAIによるカテゴリ自動分類
        
    - 外部MROカタログ（MonotaRO, Misumi等）とのパンチアウト・カタログ連携
        
- **13.3 役務の完了報告（検収代替）**
    
    - 「納品」の代わりに「作業完了報告書」などの証憑を添付
        
    - 申請者による「役務完了承認」をもって検収とするフロー
        
- **13.4 業種差分へのUI/データ構造対応**
    
    - 建設業（現場納入）、医療機器（ロット管理）など、業種特有のヘッダー項目や検収形態への切替設定
        

### 14. 異常系・例外処理

現実の業務で必ず発生する例外を処理する。

- **14.1 設変起因の例外**: （5.5 関連）
    
    - キャンセル料、特急費、金型修正費の申請・承認・集計
        
- **14.2 受入異常**: （7.4 関連）
    
    - 欠品、破損、仕様不一致の記録（写真添付）
        
    - 返品、交換、再手配のワークフロー管理
        
- **14.3 クレーム・是正処置の社内外共有**
    
    - クレーム報告とサプライヤへの是正処置依頼
        
    - 責任分担（社内、サプライヤ）の記録と対応履歴の共有
        
- **14.4 請求・支払異常**
    
    - 二重請求、誤請求の検知（8.2）と、差戻し・支払保留ステータス管理
        

### 15. AI Copilot（横断・戦略支援機能）

初期企画概要にあった「生成AI」をネイティブに組み込んだ協働機能。

- **15.1 ガイド付き購買（Guided Buying）**
    
    - **自然言語チャットボット**: 「XXが欲しい」と入力すると、AIが社内規定を検索し、正しい申請フォームや推奨カタログ品を案内（マーベリック・スペンド防止）
        
- **15.2 文書作成・要約・比較（コンテンツ生成）**
    
    - **RFx・契約書ドラフト自動生成**: AIが「XXの契約書ドラフトを作って」に応答
        
    - **仕様書・見積書の要約と比較**: 複数サプライヤの複雑な技術仕様書や見積条件の差分をAIが要約・ハイライト
        
    - 交渉サマリー（議事録）の自動生成
        
- **15.3 交渉戦略のコーチング**
    
    - （3. 見積比較 と連動）
        
    - 過去の交渉履歴、市場価格、サプライヤの状況を分析
        
    - **最適な交渉目標値（BATNA）**と代替案のアイデア出しをAIがサポート
        
- **15.4 市場調査とリサーチ（インテリジェンス）**
    
    - （9. 実績集計 と連動）
        
    - 外部の市場環境データ（市況、価格変動）と社内データを統合分析
        
    - **供給途絶リスクの早期警戒**: AIが地政学、天候、財務リスクを監視し「このサプライヤは危険」と早期に警告
