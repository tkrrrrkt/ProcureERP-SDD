# Code Review

## 目的
指定されたファイルまたはディレクトリのコードをレビューし、改善点を指摘する。

## 実行手順

1. **対象を特定**: 引数 `$1` で指定されたパスを確認
2. **コードを読む**: Read ツールでファイル内容を取得
3. **レビュー観点**:
   - コードの可読性
   - エラーハンドリング
   - セキュリティリスク
   - パフォーマンス
   - 命名規則
4. **結果を報告**: 改善点をリスト形式で提示

## 出力形式

```markdown
## レビュー結果: [ファイル名]

### 良い点
- ...

### 改善点
- [ ] 行XX: 〇〇を修正すべき
- [ ] 行YY: 〇〇の考慮が必要

### 推奨アクション
1. ...
```
